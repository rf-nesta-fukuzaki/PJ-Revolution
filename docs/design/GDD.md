## 1. コンセプト概要

### 1.1 ワンライン
2〜4人のトレジャーハンターが、ポップな古代文明の遺跡が眠る山を登り、壊れやすい遺物を物理演算で運び下ろすCo-opカオスゲーム。

### 1.2 エレベーターピッチ
「Cairnの登山 × R.E.P.O.の物理カオス × ポップな古代文明」。登るのは簡単、でも遺物を運ぶと物理演算が牙を剥く。歌う壺がボイチャを妨害し、磁力の兜がチームの装備を引き寄せ、ぬるぬるの魚像が手から滑り落ちる。協力しないと運べないが、個人スコアで競い合う「フレネミー」構造が自然な笑いと駆け引きを生む。

### 1.3 市場ポジション
「物理演算クライミング × Co-op」のヒット作はまだ存在しない。ソロ物理クライミングはCairn / A Difficult Game About Climbingが埋め、Co-op物理はR.E.P.O.が埋めたが、その交差点が空いている。ここを狙う。

### 1.4 ターゲット
- R.E.P.O. / Lethal Company / Content Warningのプレイヤー層
- 配信者・ストリーマー（物理カオスはクリップの宝庫）
- カジュアルCo-opゲーマー（操作のハードルが低い）

### 1.5 トーン＆世界観
コミカル・ポップ。謎の古代山岳文明は高度な技術を持っていたが、やることが全部おかしい。黄金のアヒル像を崇拝し、歌う壺を神器とし、魚像に油を塗る。プレイヤーは「なんでこんなもの作ったんだよ」とツッコミながら丁寧に運ぶ。

### 1.6 ビジュアルスタイル
シンプルなローポリ。デフォルメされたキャラクター。明るくポップな色彩。技術リスクを最小化しつつ、視認性とコミカルさを両立。

#### ビジュアルガイドライン
| 項目 | 仕様 |
|------|------|
| ポリゴン数目安（キャラクター） | 3,000〜5,000 tri |
| ポリゴン数目安（遺物1体） | 500〜2,000 tri |
| ポリゴン数目安（環境オブジェクト） | 100〜1,000 tri |
| カラーパレット | HSL基準。彩度60-80%、明度50-70%。暗色はHSLの明度25-35% |
| メインカラー | ティール(H:180)、コーラル(H:15)、アンバー(H:40)、ラベンダー(H:270) |
| テクスチャ解像度 | 最大512×512。フラットシェーディング推奨 |
| ライティング | Directional Light×1 + 環境光。リアルタイムシャドウ（Hard Shadow） |
| アウトライン | なし。シルエットの明快さで視認性を確保 |

## 2. コアゲームループ

### 2.1 遠征（1ラン）の流れ

1ランは約30〜40分。「ベースキャンプ → 登攀 → 遺物発見 → 運搬 → 帰還判断」を繰り返すロングラン形式。

```
ベースキャンプ（準備）
    ↓
低地帯（導入・操作習熟）
    ↓ 遺物1発見 → 運搬開始
中腹（本格化・天候悪化）
    ↓ 遺物2発見 → 運搬中のカオス
高地帯（クライマックス）
    ↓ 遺物3発見 → 最高難度の運搬
帰還判断：「もう1つ行くか、帰るか」
    ↓
下山 or ヘリ呼び → リザルト
```

### 2.2 フェーズ詳細

#### ベースキャンプ（準備フェーズ）〜2分
- チーム共有予算（100pt）でアイテムを購入
- ロッカーから物理的にアイテムを掴んでバックパックに入れる
- 「今日の天気」「ルート状況」がボードに表示（詳細はセクション10.6参照）
- 誰が何を持つかの相談（ここで自然に役割が決まる）
- 装備の投げ渡しで最初の笑いが生まれる
- **出発条件：** 全員が出発ゲート前に立ち、ホストが「出発」インタラクトを実行

#### 登攀フェーズ（メイン）〜20-30分
- 標高帯ごとに難易度がエスカレート
- 遺物を発見するたびに「持っていくか」の判断
- 天候変化、ハザード、ルート選択が常に発生
- 遺物を運搬しながらの登攀/下山がカオスのピーク
- **タイマー：** なし。時間制限はないが天候は時間経過で悪化する

#### 帰還判断
- いつでも帰還を選択可能
- **帰還方法A — 下山：** 登ったルートを逆走して下山。ハザードは依然有効。ベースキャンプに到達で帰還成功
- **帰還方法B — ヘリ呼び：** フレアガンを上空に発射（エイムを真上に向けて使用）。**60秒後にヘリが到着**（詳細はセクション2.4参照）
- **帰還投票：** 帰還したいプレイヤーがF5キーで帰還提案（詳細はセクション14.8参照）。提案なしでも個人で下山開始は可能（ただしチーム分断リスク）
- 帰還＝成功。持ち帰った遺物の状態で報酬決定
- 「もう1つ行く」はハイリスク・ハイリターン
- 欲張りすぎて全滅＝遺物ゼロで帰還

#### 下山帰還の完了条件

| 項目 | 仕様 |
|------|------|
| **帰還エリア** | ベースキャンプ出発ゲートの周囲に配置する BoxCollider (isTrigger=true, サイズ 10m×4m×10m)。Tag: `ReturnZone` |
| **帰還トリガー** | 最初の**生存者**が帰還エリアに入った瞬間、全プレイヤーのHUDに「帰還カウントダウン: 120秒」を表示開始 |
| **カウントダウン** | 120秒。全生存者が帰還エリア内に到達した時点でカウントダウン即終了 → Returning へ遷移 |
| **タイムアウト** | 120秒経過時にエリア外にいる生存者がいる場合でも、Returningへ遷移。エリア外の生存者・遺物はリザルトに**含まれない**（持ち帰れなかった扱い） |
| **遺物の持ち込み判定** | 遺物が帰還エリア内にある（OnTriggerStay）= 持ち帰り成功。プレイヤーが運搬中でなくても**エリア内の地面に置いてあればOK** |
| **幽霊の帰還** | 幽霊プレイヤーは帰還エリアの判定対象外（カウントダウンに影響しない。リザルトには自動的に含まれる） |
| **1人プレイ時** | 2人チームで1人が切断した場合、残り1人が帰還エリアに入ればカウントダウン0秒で即遷移 |

#### リザルト
- チームスコア + 個人スコアのスコアボード
- コメディ称号の授与
- コスメティック報酬の解放
- **表示時間：** 30秒間の自動演出後、任意のキーで「メインメニューへ戻る」

### 2.3 リスク・リターン設計
- 帰還できればOK（安全プレイも有効な戦略）
- 遺物の数と状態で報酬が決まる
- 高地帯のレア遺物は高価値だがリスクも高い
- 全滅してもペナルティは「今回の報酬ゼロ」のみ（メタ進行に影響なし）

### 2.4 ヘリ演出仕様

| 項目 | 仕様 |
|------|------|
| ヘリの見た目 | ローポリの民間ヘリコプター。黄色×黒のカラーリング。「RELIC RESCUE」のステッカー |
| 呼び方 | フレアガンを上空（仰角60°以上）に発射 |
| 到着時間 | フレア発射から60秒後 |
| 到着演出 | 画面外からローター音SE（段階的に大きくなる。20秒前から聞こえ始める）→ 上空からヘリが降下 → 最寄りヘリパッドポイントにホバリング |
| ダウンウォッシュ | ヘリ下方に風力30Nの下向き風（半径10m）。プレイヤー・遺物に影響あり。遺物が吹き飛ばされないよう注意 |
| 搭乗方法 | ヘリの側面ドア（ハイライト表示）にEキーでインタラクト → プレイヤーがヘリ内にテレポート（搭乗アニメーション1秒） |
| 遺物の搭載方法 | 遺物を持ったままヘリの荷台エリア（ヘリ後部。青いハイライトゾーン）に入り、Eキーで遺物をヘリに固定（遺物が消えてヘリ内UIに表示） |
| 搭乗猶予時間 | 30秒。画面上にカウントダウン表示。猶予終了でヘリ自動離陸 |
| 離陸演出 | ヘリが上昇 → カメラが山の全景を映すシネマティックカメラ（5秒）→ フェードアウト → リザルト画面 |
| 未搭乗プレイヤー | 離陸時にヘリに乗っていないプレイヤーは取り残される（下山での帰還が必要。全員搭乗でなくても離陸） |
| 全滅後のヘリ | フレア発射後に全滅した場合、ヘリは来るが誰も搭乗できず60秒後に離脱。リザルト画面（報酬ゼロ）へ |

## 3. プレイヤーキャラクター仕様

### 3.1 基本パラメータ

| パラメータ | 値 | 説明 |
|-----------|-----|------|
| HP（体力） | 100 | ゼロで死亡 |
| HP自然回復 | なし | アイテムでの回復もEA版ではなし |
| スタミナ | 100 | クライミング・ダッシュ等で消費 |
| スタミナ回復速度（静止時） | 15/秒 | 地面に立っている状態 |
| スタミナ回復速度（歩行時） | 8/秒 | |
| スタミナ回復速度（クライミング保持時） | 0/秒 | 壁に掴まっているだけで回復なし |
| 質量 | 70 kg | Rigidbody.mass |
| 高さ | 1.6 m | CapsuleCollider.height |
| 半径 | 0.3 m | CapsuleCollider.radius |
| 最大所持重量 | 6 ユニット | アイテムのみ。遺物は別枠（手持ち/背負い） |
| 最大スロット数 | 4 スロット | バックパック4枠 |

### 3.2 重量単位の定義

本GDDでは2種類の重量単位を使用する：

| 単位 | 用途 | 説明 |
|------|------|------|
| **kg（キログラム）** | 物理演算専用 | Rigidbody.massに設定する値。物理シミュレーション（衝撃力計算、落下、風の影響等）に使用 |
| **ユニット** | ゲームプレイ専用 | 所持重量上限、移動速度低下、スタミナ消費増加の計算に使用する抽象的な重さ。アイテム・遺物共通 |

> **重要：** 移動速度・スタミナへの影響は全て**ユニット**で計算する。kgは物理エンジン用の設定値であり、ゲームプレイ計算には参照しない。

### 3.3 移動パラメータ

| 状態 | 速度 | スタミナ消費 |
|------|------|-------------|
| 歩行 | 4.0 m/s | 0/秒 |
| ダッシュ | 7.0 m/s | 12/秒 |
| しゃがみ歩行 | 2.0 m/s | 0/秒 |
| クライミング移動 | 2.0 m/s | 10/秒 |
| クライミング保持（移動なし） | 0 m/s | 5/秒 |
| ジャンプ | 高さ1.5m / 距離2.0m | 15/回 |
| 遺物運搬時（手持ち） | 基本速度 × (1 - 運搬重量ユニット × 0.1) | 移動消費 × (1 + 運搬重量ユニット × 0.15) |
| 遺物運搬時（背負い） | 基本速度 × (1 - 運搬重量ユニット × 0.08) | 移動消費 × (1 + 運搬重量ユニット × 0.12) |

> **速度低下の計算例：**
> - アヒル像（運搬重量3ユニット）を手持ち歩行 = 4.0 × (1 - 3×0.1) = 4.0 × 0.7 = **2.8 m/s**
> - 大石板（運搬重量5ユニット）を1人で手持ち歩行 = 4.0 × (1 - 5×0.1) = 4.0 × 0.5 = **2.0 m/s**（最低速度）
> - 大石板を2人で手持ち = 1人あたり2.5ユニット扱い → 4.0 × (1 - 2.5×0.1) = **3.0 m/s**

**最低速度保証：** 運搬重量によらず移動速度は**最低1.0 m/s**を保証（式の結果が1.0未満なら1.0にクランプ）。

### 3.4 ダメージテーブル

| ダメージ源 | ダメージ量 | 備考 |
|-----------|-----------|------|
| 落下（3m未満） | 0 | ノーダメージ |
| 落下（3m〜10m） | (落下距離 - 3) × 8 | 最大56ダメージ |
| 落下（10m〜15m） | (落下距離 - 3) × 8 | 最大96ダメージ |
| 落下（15m以上） | 即死（100） | キルフォール |
| キルプレーン | 即死（100） | 各ゾーンの最低標高 -30m に設置 |
| 落石 | 30〜50（岩サイズ依存） | |
| 凍傷（吹雪ゾーン・シェルターなし） | 5/秒 | ゾーン5-6で吹雪中 |
| 高山病（標高2000m以上・酸素なし） | 3/秒 + 移動速度-30% + 視界ぼやけ | ゾーン5-6 |
| プレイヤー間衝突 | 0 | ダメージなし（押し合いは可能） |
| アイテムぶつけ | 5〜15（アイテム質量依存） | 味方に当てた場合 |

### 3.5 スタミナ切れ時の挙動
- スタミナ0でクライミング中 → **グリップ解除**。掴みポイントから手が離れ滑落
- スタミナ0でダッシュ中 → 歩行速度に強制移行。**2秒間ダッシュ不可**
- スタミナ0のプレイヤーはHUDのスタミナバーが**赤く点滅**

### 3.6 プレイヤー名

| 項目 | 仕様 |
|------|------|
| 取得方法 | Steam表示名を自動使用（Steamworks APIの`SteamFriends.GetPersonaName()`） |
| 手動変更 | 設定画面で上書き可能（ゲーム内表示名のみ。Steam名は変わらない） |
| 文字数制限 | 2〜16文字 |
| 使用可能文字 | Unicode全般（絵文字含む） |
| 表示箇所 | ロビー、HUDチームメイト表示、リザルト、ピンマーカー |

## 4. 操作システム

### 4.1 ハイブリッド方式
「登り簡単 × 運搬カオス」のハイブリッド操作。

### 4.2 キーバインド一覧

| 操作 | キー（M&K） | ゲームパッド | 説明 |
|------|------------|-------------|------|
| 移動 | WASD | 左スティック | 地上・壁面ともに |
| カメラ操作 | マウス移動 | 右スティック | |
| ジャンプ | Space | A / × | 地上のみ |
| ダッシュ | Shift（ホールド） | L3 押し込み（トグル） | スタミナ消費 |
| しゃがみ | Ctrl（トグル） | R3 押し込み（トグル） | |
| 掴む / 離す | 左クリック | LT / L2 | クライミングポイント、アイテム、遺物（文脈依存。4.7参照） |
| 投げる | 右クリック（ホールド→リリース） | RT / R2 | ホールド中にパワーゲージ蓄積 |
| インタラクト | E | Y / △ | ショップ購入、ヘリ搭乗、祠復活等 |
| アイテム使用 | R | X / □ | 手持ちアイテムの固有アクション |
| アイテムドロップ | G | D-pad Down | 手持ちアイテム/遺物を足元に丁寧に置く |
| インベントリ開閉 | Tab | Menu / Options | バックパック画面 |
| ピン（マーカー） | F | RB / R1 | ワールド上にマーカー設置（4.9参照） |
| ロープ切断 | X | LB / L1 | 自分に繋がっているロープを切断 |
| プッシュトゥトーク | V（ホールド） | D-pad Up（ホールド） | ボイスチャット |
| 帰還提案 | F5 | D-pad Left + Menu | 帰還投票を開始（14.8参照） |
| クイックスロット | 1〜4 | D-pad 方向 | バックパック内アイテム切替 |
| エモート | T | D-pad Right | エモートホイール呼出（4.10参照） |
| 一時停止 | Esc | Menu / Options | 設定・離脱メニュー |

> **キーリバインド：** 設定画面にて全キーのリバインドが可能。ゲームパッドはプリセット3種（デフォルト/代替A/代替B）から選択。

### 4.3 登攀操作（ポイント＆グラブ方式）
- 通常移動：TPS標準コントローラー操作（WASD + マウス）
- **クライミングポイント検出：** プレイヤーから半径2.0m以内のクライミングポイントを検出
- **ハイライト表示：** 検出されたポイントは白い円形アウトラインが表示。最も近いポイントは黄色ハイライト
- 左クリックで掴む → 掴み中は左クリックホールド維持 → リリースで離す
- **同時掴み：** 片手1ポイントのみ。両手で別ポイントは不可
- スタミナ管理が登攀の難しさ
- 直感的で初心者もすぐ遊べる

#### クライミングポイントの技術定義

| 項目 | 仕様 |
|------|------|
| **識別方法** | Tag: `ClimbingPoint`。Layer: `Interactable`（レイヤー10） |
| **コンポーネント** | `ClimbingPoint.cs`（MonoBehaviour）+ SphereCollider（isTrigger=true, radius=0.3m） |
| **見た目** | 岩の突起/棚/割れ目を模したメッシュ。掴み可能範囲に半透明の円形アウトライン（シェーダーで動的表示） |
| **検出方式** | `Physics.OverlapSphere(playerPos, 2.0f, interactableLayerMask)` で毎フレーム検出 |
| **ハイライト色** | 未選択: 白（Alpha 0.5）/ 最近接: 黄色（Alpha 0.8）/ 遺物保持中: 非表示 |
| **IK Target** | 各ポイントにIK用のTransform（手の位置・回転を定義）を子オブジェクトとして保持 |

#### 動的ポイント（アイテムで生成されるポイント）

| 生成元 | 仕様 |
|--------|------|
| **アイスアックス** | R使用で壁面Raycast → 着弾点にClimbingPoint Prefabを動的生成。耐久消費あり(4/回) |
| **グラップリングフック** | R使用で射出 → 着弾点（15m以内の壁面/天井）にClimbingPoint Prefabを動的生成 |
| **アンカーボルト** | R使用で壁面に設置 → ロープ固定点として機能。ClimbingPointではないがロープ接続可能 |
| **ネットワーク同期** | 動的生成はServerRpcで要求。ホストがNetworkObject.Spawn()で全クライアントに同期 |
| **永続性** | 動的ポイントは遠征終了まで消滅しない。ただしアイスアックスポイントは生成から5分で消滅（劣化演出） |

#### バックパック格納フロー

| 操作 | 手順 | 詳細 |
|------|------|------|
| **格納** | アイテムを手に持った状態 → Tabキー → インベントリUIが開く → 空きスロットをクリック → アイテムがスロットに格納 | スロット/重量が不足している場合は赤いエラーテキスト「スロット不足」or「重量超過」を2秒表示 |
| **取り出し** | Tabキー → インベントリUI → スロット選択 → アイテムを手に持つ | 手に既にアイテムを持っている場合は入れ替え（手のアイテムがスロットへ、選択スロットのアイテムが手へ） |
| **クイック取出** | 1〜4キー → 対応スロットのアイテムを即座に手に持つ | Tabキーを開かず直接切替。運搬中は使用不可 |
| **バックパック満杯時** | 手にアイテムを持った状態で拾おうとする → 「バックパックがいっぱいです」プロンプト表示 | 手のアイテムを先にスロットに入れるか、ドロップする必要がある |

### 4.4 運搬操作（物理ゴリゴリ方式）
- 遺物を左クリックで掴む → **運搬モード**に移行
- 遺物を持つと重心が変わり、バランスを取る必要がある
- **重心バランス：** 自動補正なし。遺物の位置・傾きに応じてプレイヤーの移動方向に力が加わる。プレイヤーは「反対方向に移動入力する」ことで手動バランス
- **バランス表示：** HUDに傾斜インジケーター（丸い水準器アイコン）を表示。中央＝安定、外周＝転倒リスク
- 担架/大型遺物は2人で持つ（両端の高さ差が**45度以上**で遺物が滑り落ちる）
- 坂道で加速、風で煽られる、ぶつけると破損
- **1人が同時に持てる遺物は1個まで**（両手がふさがる）。ただし固定ベルト装着中の背負い遺物は手持ちとは別枠

### 4.5 投擲メカニクス
- 右クリックホールドで**パワーゲージ**蓄積（0〜100%、2秒で最大）
- ゲージはクロスヘア周囲に円弧で表示
- リリースで投擲。投擲力 = `baseForce(5N) + gaugePct × maxForce(20N)`
- 投擲方向 = カメラ正面方向
- **軌道プレビュー：** ホールド中に点線の放物線ガイド表示（3点予測）

### 4.6 遺物運搬とクライミングの同時操作

**運搬モードとクライミングモードは相互排他。** 遺物を手に持ったまま壁を登ることはできない。これが本ゲームのコアチャレンジ。

#### 運搬状態の分類と操作制限

| 運搬状態 | 手の状態 | クライミング | ジャンプ | ダッシュ | 速度低下 |
|---------|---------|------------|---------|---------|---------|
| **手ぶら** | 両手空き | ○ | ○ | ○ | なし |
| **手持ち運搬**（片手/両手） | 片手or両手使用中 | **×** | × | × | あり（重量依存） |
| **背負い運搬**（固定ベルト使用） | 両手空き | **○**（スタミナ消費+50%） | ○ | × | あり（軽度） |
| **担架運搬**（2人） | 両手使用中 | **×** | × | × | あり |

#### 遺物を崖の上に運ぶための典型的プレイフロー

```
1. 遺物を崖の手前まで手持ちで運ぶ
2. 遺物を崖の足元に置く（Gキー）
3. プレイヤーAが壁を登って崖の上へ
4. プレイヤーBが下で遺物を見守る
5. 方法を選択：
   a) プレイヤーAがロープを下ろす → 遺物にロープを括り付け（E）→ Aが引き上げ
   b) ポータブルウインチをAが設置 → 下のBが遺物にケーブル接続 → 巻き上げ
   c) 固定ベルトで小型遺物を背負い、そのまま登攀（重量ペナルティあり）
   d) 坂道の場合：2人で担架に載せて押して登る
```

> **設計意図：** 「遺物を持ったまま登れない」ことが、チーム協力の必然性とカオスの源泉。ロープやウインチの活用が攻略の鍵となり、「遺物を引き上げている最中に天候が悪化」「ウインチのケーブルが切れて遺物が崖下に」等のドラマが自然発生する。

#### 遺物のロープ括り付け

- ロープを持った状態で遺物に近づく（2m以内）→ Eキーで遺物にロープを括り付け
- 括り付けた遺物はロープの端に物理接続（ConfigurableJoint）
- ロープの反対端を持ったプレイヤーが引き上げる（通常のロープ張力で物理移動）
- **引き上げ中の遺物ダメージ：** 壁面に擦れるため、衝突ダメージが断続的に発生。梱包キットで軽減推奨

### 4.7 左クリックの文脈依存優先度

左クリックは複数の機能を持つため、以下の優先度ルールで動作する：

| 優先度 | 現在の状態 | 左クリックの挙動 |
|--------|----------|----------------|
| 1（最高） | 遺物/アイテムを手に持っている | **リリース**（手から離す。物理落下） |
| 2 | 壁に密着 + 手が空いている + クライミングポイント検出中 | **クライミングポイントを掴む** |
| 3 | 地面 + 手が空いている + アイテム/遺物が近くにある | **アイテム/遺物を拾う** |

> **競合しないケース：** 遺物を持っている状態で壁に近づいても、クライミングポイントのハイライトは**表示されない**（「持ったまま登れない」ことを視覚的に伝達）。代わりに「Gキーで遺物を置く」プロンプトが表示される。

### 4.8 遺物を「置く」操作の詳細

| 操作 | キー | 挙動 | 遺物へのダメージ |
|------|------|------|----------------|
| **丁寧に置く** | G | 足元にゆっくり下ろすアニメーション（1秒）。遺物を地面にそっと設置 | 0（ノーダメージ） |

| **投げる** | 右クリックチャージ→リリース | 投擲メカニクスで投げる | 着弾時の衝撃ダメージ |

## 5. 物理演算システム

### 5.1 ロープ物理

| パラメータ | ショートロープ | ロングロープ |
|-----------|--------------|------------|
| 長さ | 10m | 25m |
| セグメント数 | 20 | 50 |
| 実装方式 | ConfigurableJoint チェーン | 同左 |
| 弾性係数（Spring） | 500 | 400 |
| 減衰係数（Damper） | 50 | 40 |
| 最大張力（切断閾値） | 3000N | 2500N |
| たるみ | セグメント間のGravity適用 | 同左 |
| 引っかかり | コライダー接触で摩擦発生 | 同左 |

- プレイヤー同士を繋ぐ（任意の2人を選択して接続）
- **接続方法：** ロープ所持者が他プレイヤーに近づき（2m以内）Eキーでインタラクト
- **遺物への接続：** ロープ所持者が遺物に近づき（2m以内）Eキーで括り付け（セクション4.6参照）
- 1人が滑落すると全員引っ張られる（張力がJoint経由で伝播）
- **切断：** ロープに繋がれたプレイヤーがXキーで即切断。切断されたロープは物理オブジェクトとして落下
- 切断後のロープは再使用不可（消滅しない。物理オブジェクトとして残り続ける）
- **ロープの耐久消費：** ロープが壁面/地面に接触している間、`接触力 / 500` の耐久消費が毎秒発生（長時間の擦れで徐々に劣化）

### 5.2 遺物ダメージ計算

```
relicDamage = impactForce / relicToughness × damageMult

where:
  impactForce = contactPoint.impulse.magnitude (Newton)
  relicToughness = 遺物ごとの固有値（後述テーブル）
  damageMult = 地面接触: 1.0 / 壁接触: 0.8 / プレイヤー接触: 0.5
```

> **damageMult 全リスト：** 地面接触: 1.0 / 壁接触: 0.8 / プレイヤー接触: 0.5 / **遺物同士の衝突: 0.7**（双方にダメージ適用。衝撃力は共有）

| 衝撃力(N) | 参考シチュエーション |
|-----------|-------------------|
| 0〜50 | 軽い揺れ、歩行振動 → ほぼノーダメージ |
| 50〜200 | 軽く落とす（1m程度） |
| 200〜500 | 中程度の落下（3m程度） |
| 500〜1000 | 高所落下、壁への激突 |
| 1000以上 | 崖から転落 → 致命的ダメージ |

#### 遺物状態と報酬係数（正式定義）

以下の**段階式テーブルが報酬計算の正式な定義**。スコアリング（セクション12.1）もこのテーブルを参照する。

| 状態 | HP範囲 | 報酬係数 | 見た目変化 |
|------|--------|---------|-----------|
| 完品 | 100% | ×1.0 | 変化なし |
| 軽傷 | 70〜99% | ×0.85 | 小さなヒビのデカール表示 |
| 損傷 | 50〜69% | ×0.6 | ヒビ拡大 + 欠けパーティクル |
| 大破 | 1〜49% | ×0.25 | 大きなヒビ + 破片散乱 |
| 破壊 | 0% | ×0 | 粉砕アニメーション → オブジェクト消滅 |

#### 遺物凍結ダメージ（ゾーン5-6）

| 項目 | 仕様 |
|------|------|
| 発生条件 | 吹雪中にサーマルケースに入っていない遺物が露出している場合 |
| ダメージ量 | 2pt/秒（遺物HPに対して） |
| 対象遺物 | 全遺物共通 |
| 視覚表現 | 遺物表面に霜のデカールが段階的に広がる。完全凍結（30秒経過）で白いパーティクル |
| 無効化方法 | サーマルケースに格納 / ビバークテント内に置く / 屋内に持ち込む |

### 5.3 天候の物理パラメータ

| 天候 | 風力(N) | 風向き変動 | 摩擦変化 | 視界距離 | 発生条件 |
|------|---------|-----------|---------|---------|---------|
| 晴れ | 0〜5 | — | ×1.0 | 無制限 | デフォルト。ゾーン1-2で頻出 |
| 曇り | 5〜15 | 30秒ごとに45°回転 | ×1.0 | 200m | 晴れから移行 |
| 霧 | 0〜5 | — | ×1.0 | 30m | ゾーン3-4で発生 |
| 雨 | 10〜30 | 15秒ごとに90°回転 | ×0.7 | 80m | ゾーン3-4で発生 |
| 吹雪 | 30〜60 | 10秒ごとにランダム | ×0.5 | 15m | ゾーン5-6専用 |

- **風力の適用：** 各Rigidbodyに `windDirection × windForce × (surfaceArea / 1.0)` の力を毎FixedUpdateで適用
- **surfaceArea：** プレイヤー = 1.0、遺物 = 各遺物テーブル参照
- **天候サイクル：** ゲーム内時間で5分ごとに天候判定（現天候の遷移確率テーブルで決定）
- **天候遷移確率：**

| 現在 → | 晴れ | 曇り | 霧 | 雨 | 吹雪 |
|--------|------|------|-----|-----|------|
| 晴れ | 40% | 40% | 15% | 5% | 0% |
| 曇り | 20% | 30% | 20% | 25% | 5% |
| 霧 | 15% | 25% | 30% | 25% | 5% |
| 雨 | 10% | 25% | 15% | 35% | 15% |
| 吹雪 | 0% | 5% | 5% | 20% | 70% |

> **高地帯補正（ゾーン5-6）：** 吹雪列の確率に+20%を加算し、その分を他の列から按分で減算して合計100%に正規化する。
> 例：曇り行のゾーン5-6 → 吹雪25%に変更 → 残75%を他4列の元比率（20:30:20:25 = 95%）で按分 → 晴れ15.8%, 曇り23.7%, 霧15.8%, 雨19.7%

### 5.4 装備投げ渡し
- 全アイテムが物理オブジェクト（Rigidbody + Collider）
- 投げて渡す、落とす、壊す
- バックパックからの出し入れ：Tabキーでインベントリ画面 → スロット選択 → 手に持つ

### 5.5 滑落時の挙動
1. クライミング中にスタミナ切れ or グラブリリース → **掴みポイントから離脱**
2. 離脱後 **0.5秒間** は再グラブ猶予（この間に左クリックで近くの掴みポイントを掴める）
3. 猶予時間経過 → **通常落下**（プレイヤーは制御可能。空中移動入力で水平方向にわずかに移動可能: 1.5m/s）
4. ロープ接続中の場合 → ロープ長分だけ落下した後、張力でぶら下がり状態に
5. **Ragdoll化条件：** 落下中に壁面/地面に**衝突速度15m/s以上**で接触 → 3秒間Ragdoll → 制御復帰 + 落下ダメージ適用

| Physics.defaultSolverIterations | 8 | 標準6から引き上げ。ロープ50セグメントのJointチェーン安定化のため |
| Physics.defaultSolverVelocityIterations | 2 | 標準1から引き上げ。衝撃伝播の精度向上 |
| Physics.gravity | (0, -9.81, 0) | Unity標準 |
| Physics.bounceThreshold | 2.0 | 標準2.0。衝撃力2N未満のバウンスを無視 |

## 7. 死亡＆幽霊システム

### 7.1 死亡条件
- HP（体力）がゼロ：落下ダメージ、落石、凍死等
- キルプレーンへの到達：各ゾーンの最低標高 -30m 地点に設置された不可視プレーン

### 7.2 死亡時の処理
1. プレイヤーのRigidbody → Ragdoll化（3秒間の死亡アニメーション）
2. 所持アイテム全てがドロップ（物理オブジェクトとして死亡位置に出現）
3. 運搬中の遺物はその場にドロップ（物理継続）
4. 3秒後 → 画面が暗転（1秒フェード） → 幽霊モードに移行

### 7.3 偵察幽霊システム
- 死亡したプレイヤーは透明な幽霊になる
- **幽霊の見た目：** 半透明（Alpha 0.3）のプレイヤーモデル。青白い発光エフェクト
- 物理的な干渉は不可（全コライダーをTriggerに切替、Rigidbody.isKinematic = true）
- **移動方式：** フリーフライ（WASD + Space上昇 + Ctrl下降）。速度 8.0m/s
- **壁抜け：** 可能（物理干渉なしのため）
- **移動範囲制限：** 生存者の最遠プレイヤーから半径200m以内
  - **制限到達時の処理：** 半径180mを超えると移動速度が段階的に減衰（180m: 6m/s, 190m: 3m/s, 200m: 0m/s）。200mで完全停止。見えない壁ではなく速度減衰方式で自然に制限
  - **HUD表示：** 180mを超えると画面端に「範囲制限に近づいています」警告テキスト表示（赤点滅）
- できること：
  - ピン（マーカー）を打てる（Fキー。最大5個同時設置）
  - ボイチャで指示できる（プロキシミティは適用される。幽霊からの声は**エコーエフェクト付き**）
  - 「上から見た偵察役」として貢献
- 復活条件：山に散らばっている「祠」を幽霊状態で見つけてEキーで3秒間インタラクト（チャネリング）
  - 中断可能（リトライ可）
  - 復活後はHP50、スタミナ100で祠の位置に出現（ビバークテント設置済みの場合はテント位置も選択可能）
  - 1プレイヤーにつき**1回のみ**復活可能（2回目の死亡は遠征終了まで幽霊）
- 幽霊状態でも偵察ボーナスで少し貢献度が入る

#### ポータブルウインチ — 詳細操作フロー

| ステップ | 操作 | 処理 |
|---------|------|------|
| 1. 設置 | ウインチを手に持った状態 → Rキー | 足元の地面にウインチを設置。設置にはほぼ水平な地面が必要（傾斜30°以下）。設置後はウインチはワールドに固定（Rigidbody.isKinematic = true） |
| 2. ケーブル展開 | 設置済みウインチにEキー | ウインチからケーブル端が出現（物理オブジェクト。ロープと同様のConfigurableJoint チェーン。セグメント数20。最大長20m） |
| 3. ケーブル接続 | ケーブル端を持った状態で遺物/プレイヤーに近づき → Eキー | 対象にJoint接続。接続SE再生。遺物の場合はRelicGrabPointに接続 |
| 4. 巻き上げ開始 | 設置済みウインチに近づき（3m以内）→ Rキー | 巻き上げ開始。ケーブルのJoint間距離を毎FixedUpdateで短縮（1.5m/s）。稼働中は耐久を5/秒消費。巻上SE（winch_loop）ループ再生 |
| 5. 停止 | 巻き上げ中にRキー or ウインチから3m以上離れる | 巻き上げ停止。耐久消費停止。ケーブル長は停止時点で固定 |
| 6. 回収 | 設置済みウインチにGキー | ウインチを回収して手に持つ（ケーブル接続中は回収不可。先にXキーでケーブル切断が必要） |
| 7. ケーブル切断 | ウインチに繋がれたプレイヤーがXキー or 張力5000N超過 | ケーブル物理切断。切断音SE。ケーブルは残骸として物理オブジェクト化 |

> **操作主体：** ウインチは**誰でも操作可能**（設置者に限定しない）。設置後は3m以内の任意のプレイヤーがR/Gで操作できる。
> **ネットワーク同期：** 設置/巻上げ開始/停止はServerRpc。ケーブルの物理演算はホスト権威。

#### 折りたたみ担架 — 物理仕様

| パラメータ | 値 |
|-----------|-----|
| **Joint構成** | 担架の両端にConfigurableJoint × 2個。各JointのConnectedBodyはプレイヤーの手ボーン |
| **担架形状（展開時）** | BoxCollider（2.0m × 0.8m × 0.15m）。遺物を上に載せる面がある |
| **担架形状（折畳時）** | BoxCollider（0.5m × 0.3m × 0.3m）。バックパック格納可 |
| **展開/折畳** | Rキー長押し1秒。展開中はアニメーション再生（コライダー即座にサイズ変更。Lerp不要） |
| **遺物の固定** | 担架上面に遺物を置くと摩擦力のみで保持（Joint固定なし。物理的にフリー）。担架のPhysicMaterialはfriction=0.8（高摩擦で安定） |
| **遺物スリップ判定** | 担架のTransform.up と Vector3.up の角度が45°以上 → 遺物が摩擦を超えて滑り落ちる（担架のPhysicMaterial.frictionを一時的に0.05に変更） |
| **2人の移動統合** | 各プレイヤーの移動入力は個別にRigidbodyに適用。担架はJoint経由で両端から引かれる。片方が速く歩くと傾く（これが45°スリップの原因） |
| **1人引きずり** | 1人だと片端のJointのみ → 反対端が地面を引きずる。引きずりの地面接触で遺物に1ダメージ/秒 |
| **張力限界** | Joint.breakForce = 4000N。超過でJointが外れ、担架がプレイヤーの手から離れる |

### 8.4 「小型」遺物の定義（固定ベルト/サーマルケース対象）
以下のサイズ「小」の遺物が固定ベルト・サーマルケースの対象：
- 古代クリスタルの杯（小/軽）
- 鎖付き双子像の片方（小×2/中だが個々は小）
- 磁力の兜（小/重）

> **浮遊する球体が固定ベルト対象外の理由：** サイズ「中」のため対象外。また、反重力で常に浮き上がる力が働いており、ベルトで固定しても浮力で外れてしまう設定。

### 8.5 ショップの同時購入管理

| 項目 | 仕様 |
|------|------|
| 同時アクセス | 全員が同時にショップUIを開ける |

### 10.7 技術実装方針
- 固定地形：Unity Terrain + ProBuilder。AI生成後に必要に応じて微調整
- ランダムレイヤー：スポーンポイントシステム。各ゾーンに事前配置したスポーンポイントをゲーム開始時にランダムアクティベート
- 天候：Particle System + Post Processing。風はRigidbodyへのForce適用
- **レイヤー確定タイミング：** ホストがロビーで「出発準備」を押した時点で全5レイヤーのシードを決定。同一シードで全クライアント同期

## 11. マルチプレイヤー設計

### 11.1 接続方式
- ルームコード制（フレンドと遊ぶ前提）
- パブリックマッチメイキングなし（EA時点）
- ホストがルーム作成 → コード共有 → 参加

#### ルームコード仕様
| 項目 | 仕様 |
|------|------|
| 形式 | 英数字大文字6桁（例: A3F7K2） |
| 文字種 | A-Z, 0-9（紛らわしい文字 I/O/0/1 は除外 → 32文字種） |
| 有効期限 | セッション中のみ有効。ホスト切断で無効化 |
| コピー機能 | コード表示横に「コピー」ボタン。クリックでクリップボードにコピー |

### 11.2 ゲームモード

#### Co-opモード（メイン）
- 2〜4人チームで1つの遠征
- 協力しないと遺物を運べない設計
- 個人貢献度スコアで報酬内の配分が変わる

#### 競争モード（サブ）— EA後アップデートで追加
- 2チーム対抗レスキューレース
- 同じ山で先に多くの遺物を持ち帰ったチームが勝ち
- 妨害あり/なし設定可能

### 11.3 ネットワーク仕様

| 項目 | 仕様 |
|------|------|
| フレームワーク | Unity Netcode for GameObjects |
| トポロジ | ホスト型P2P（サーバーコスト不要） |
| 物理Authority | ホスト権威方式（全物理演算をホストで計算） |
| 同期周波数 | 20Hz（物理同期）/ 10Hz（プレイヤーステート同期） |
| 補間方式 | クライアント側で受信間隔の線形補間（Lerp） |
| レイテンシ許容値 | 150ms以下推奨。300ms以上で警告UI表示 |
| 大型遺物の同期 | 2人で持つ遺物はホスト側で一元計算。各クライアントの入力をホストに送信→ホストが物理結果を全クライアントに配信 |

#### ホスト切断時の処理
1. ホスト切断検出（3秒間の無応答でタイムアウト）
2. **ホストマイグレーションなし**（EA版の制約）
3. 全クライアントに「ホストが切断しました」ダイアログ表示
4. 「メインメニューに戻る」ボタンのみ。進行状況は保存されない

#### クライアント切断時の処理
1. 切断検出（5秒間の無応答）
2. 切断プレイヤーのキャラクターはその場に3秒間立ったまま → 消滅
3. 所持アイテム・運搬中の遺物はドロップ
4. **再接続：** 切断から60秒以内にルームコードで同ルームに再参加可能。復帰位置は切断前の位置。60秒経過で枠解放
5. 他プレイヤーには「[プレイヤー名]が切断しました」通知

### 11.4 プロキシミティボイスチャット

| 項目 | 仕様 |
|------|------|
| 実装方式 | Dissonance Voice Chat（Unityアセットストア） |
| 入力方式 | プッシュトゥトーク（Vキー / D-pad Up） |
| 最大聴取距離 | 30m |
| 距離減衰曲線 | リニア。0m = 音量100% → 30m = 音量0% |
| 壁越しの減衰 | 壁1枚あたり-30%追加減衰（Raycastで判定） |

## 12. スコアリング＆報酬

### 12.1 チームスコア（全員共通）

```
teamScore = Σ(relicBaseValue × stateMultiplier) × survivalBonus

where:
  relicBaseValue = 各遺物の基準報酬額（セクション9.4参照）
  stateMultiplier = セクション5.2「遺物状態と報酬係数」テーブルを参照（段階式）
    完品(100%): ×1.0 / 軽傷(70-99%): ×0.85 / 損傷(50-69%): ×0.6 / 大破(1-49%): ×0.25 / 破壊(0%): ×0
  survivalBonus = 全員生還: 1.2 / 1人以上死亡: 1.0
```

#### 計算例
> アヒル像（完品 300×1.0=300）+ クリスタル杯（HP60/80=75%→軽傷→×0.85、1500×0.85=1275）+ 全員生還ボーナス
> → (300 + 1275) × 1.2 = **1890pt**

### 12.2 個人スコア（競争要素）

各プレイヤーの個人貢献度に基づき、チームスコアの報酬配分比率を決定。

| 貢献アクション | ポイント | 記録条件 |
|---------------|---------|---------|
| 遺物発見 | 50pt/個 | 遺物に最初に接触したプレイヤー |
| 遺物運搬距離 | 1pt/5m | 遺物を持っている間の移動距離 |
| ロープ設置 | 15pt/回 | ロープ接続 or アンカーボルト設置 |
| チームメイト救出 | 30pt/回 | ロープで滑落中のプレイヤーを引き上げ（下記判定ロジック参照） |
| 偵察ピン使用（幽霊） | 10pt/回 | 幽霊状態でのピン設置 |
| ハザード回避リード | 20pt/回 | ピン打ち後3秒以内にチームメイトがハザード回避（下記判定ロジック参照） |
| 食料提供 | 5pt/回 | 他プレイヤーに食料を投げて渡した |
| 遺物引き上げ | 25pt/回 | ロープ/ウインチで遺物を引き上げた |

#### 個人スコア — 判定ロジック詳細

| 判定項目 | 具体的なロジック |
|---------|---------------|
| **チームメイト救出** | 条件①: ロープ/ウインチケーブルで接続された2人のうち、一方が `!IsGrounded && velocity.y < -3` の滑落状態。条件②: ロープ張力が発生（Joint.currentForce.magnitude > 100N）。条件③: 滑落者のY座標が3秒以内に上昇に転じた（ΔY > 0.5m）。①②③を全て満たした場合、ロープ上端のプレイヤーに救出判定 |
| **チームメイトを落とした** | 条件①: プレイヤーAがプレイヤーBに物理接触（OnCollisionEnter、impulse > 50N）。条件②: 接触後5秒以内にBが `!IsGrounded` かつ落下ダメージ20以上を受けた。①②を満たした場合Aにペナルティ。**ロープ切断の場合:** 切断者が明確なのでXキーを押したプレイヤーに判定 |
| **遺物運搬距離（共同運搬時）** | 遺物を掴んでいる**全プレイヤーに均等分割**して加算。2人で掴んでいる場合、5m移動で各自0.5pt |
| **ハザード回避リード** | 条件①: プレイヤーAが「危険」ピン（赤ピン）を設置。条件②: ピン設置後3秒以内に、ピン位置の半径10m以内でハザード（落石/トラップ）が発動。条件③: その範囲内にいたチームメイトBが被弾しなかった（ハザードのダメージ判定にBが含まれなかった）。①②③でAに判定 |

### 12.3 ペナルティ

| ペナルティアクション | 減算ポイント | 記録条件 |
|-------------------|-------------|---------|
| 装備喪失 | -10pt/個 | アイテムが回収不能位置に落下 or 破壊 |
| 遺物へのダメージ | -2pt/ダメージpt | 持っている遺物にダメージを与えた |
| チームメイトを落とした | -15pt/回 | ロープ切断 or 押して落下の判定（上記「判定ロジック詳細」参照） |
| 遺物破壊 | -50pt/個 | 持っていた遺物のHPが0になった |

### 12.4 報酬配分計算

```
rawScore = max(playerIndividualScore, 0)   ← 負の場合は0にクランプ
playerShare = rawScore / Σ(allPlayerRawScores)
playerReward = teamScore × playerShare

最低保証: playerShare は最低 10%（4人チームの場合。貢献0でも10%はもらえる）
全員rawScore=0の場合: 均等配分（各自25%）
```

### 12.5 リザルト称号（コメディ）

全プレイヤーに1つ以上の称号を付与。遠征中の統計データから自動判定。

| 称号名 | 判定条件 |
|--------|---------|
| 「重力の友」 | 落下回数が最多のプレイヤー |
| 「装備マイスター」 | 装備を最も多く失ったプレイヤー |
| 「遺物クラッシャー」 | 遺物に最もダメージを与えたプレイヤー |
| 「声量キング」 | ボイチャ音量の累積が最大（マイク入力のRMS値を累積） |
| 「影のMVP」 | 幽霊状態で最も長時間偵察 or ピン数最多 |
| 「鉄人ハンター」 | 一度も死亡せずに帰還 |
| 「遺物マスター」 | 最も多くの遺物を運搬したプレイヤー |
| 「ロープの達人」 | ロープ設置/救出の合計が最多 |

> **称号の優先度：** 複数条件を満たす場合、よりレアな称号を優先。全プレイヤーが最低1つ取得するよう、フォールバック称号「参加賞：お疲れ山でした」を用意
> **タイブレーク：** 同条件のプレイヤーが複数いる場合、**その称号を全員に付与**する（排他ではない）。誰にも付かない称号が出ないよう、判定は「全員が条件値0の場合はその称号をスキップ」する（例：全員落下0回の場合「重力の友」は不付与）

## 13. コメディ設計

### 13.1 システムとして笑いを生む仕掛け

#### 遺物カオス
- 担架の両端の高さが違うと遺物が滑り落ちる
- 坂道で遺物が転がり出して追いかけっこ
- 「丁寧に運びたいのに物理が許さない」が笑いの源泉
- 遺物をロープで引き上げ中に壁にぶつかって破損

#### ロープ連鎖事故
- 1人の滑落で全員引っ張られる
- 「切れ！ロープ切れ！」「いやお前が登れ！」の自然発生

#### 装備受け渡し物理
- 崖越しに投げ渡そうとして谷底に落とす
- 仲間の顔面に当たってよろける（5ダメージ）

#### 古代文明のボケ
- 遺物の意味不明さ（「なぜアヒルを崇拝する」）
- 遺跡の石碑に書いてある文章がくだらない
- トラップのデザインがおかしい

#### リザルトの称号
- ネガティブ称号で笑いを取る
- 全員がなにかしらの称号をもらう

### 13.2 配信映えポイント
- プロキシミティボイチャ × 距離 → 「聞こえない」パニック
- 遺物の特殊挙動 → 予測不能なカオス
- ロープ連鎖 → ドミノ倒し的な全滅
- 幽霊の偵察 → 「そっちじゃない！」の空振り指示

### 14.6 リザルト画面

リザルト画面は以下の6段階で自動演出される。各段階は時間経過で自動進行。

| 段階 | 表示内容 | 時間 | 演出 |
|------|---------|------|------|
| 1. チームスコア | 画面中央に大きな数字でチームスコアをカウントアップ | 3秒 | 数字が0から高速カウントアップ → 最終値で停止 + 効果SE |
| 2. 遺物一覧 | 持ち帰った遺物の3Dモデル + 状態ラベル + 獲得ポイント | 5秒 | 遺物が1つずつスライドイン。状態に応じた色枠（完品:金 / 軽傷:銀 / 損傷:銅 / 大破:灰） |
| 3. 個人スコアボード | 各プレイヤーの個人スコア内訳テーブル | 5秒 | 1人ずつ行が追加。貢献アクション別のポイント内訳表示 |
| 4. 称号授与 | 各プレイヤーに称号カードが飛び出し | 10秒 | 1人ずつ称号カードがポップアニメーション + ファンファーレSE。カードは金枠 + 称号名 + 判定理由テキスト |
| 5. コスメ解放 | 新しいコスメが解放された場合のみ | 3秒 | 「NEW!」バッジ + コスメの3Dプレビュー + 「○○が解放されました！」テキスト |
| 6. ボタン表示 | 「メインメニューに戻る」ボタン | — | 段階5終了後に表示。任意のタイミングで押下可能 |

> **全滅時のリザルト：** チームスコア=0。段階1で「0pt」表示 + 「全滅…」テキスト。遺物一覧は空。称号は全員に「参加賞：お疲れ山でした」のみ。
> **スキップ操作：** 各段階の演出中に**Spaceキー / Aボタン**で次の段階にスキップ可能。段階1〜5の合計演出時間（約26秒）が終了後、段階6の「メインメニューへ」ボタンが表示される。ホストがボタン押下で全員遷移。

### 15.1 BGM

| シーン/状態 | コンセプト | テンポ | 楽器イメージ |
|------------|----------|-------|------------|
| タイトル/メニュー | 冒険の始まり。ワクワクする穏やかさ | BPM 100 | アコースティックギター、軽いパーカッション |
| ベースキャンプ | 準備の高揚感。軽快 | BPM 110 | マリンバ、フルート |
| ゾーン1-2（低地帯） | 探検の楽しさ。明るく軽快 | BPM 115 | 木管、弦（ピチカート） |
| ゾーン3-4（中腹） | 緊張感の上昇。テンション上がる | BPM 125 | 弦（トレモロ）、太鼓 |
| ゾーン5-6（高地帯） | 極限の緊迫感。荘厳 | BPM 90 | 低音弦、ホルン、環境音多め |

| 全滅 | 悲壮 → コミカルオチ | — | ドラムロール → チーン（拍子木） |

| 幽霊モード | 静かで少し不思議。エコー感 | BPM 80 | リバーブの効いたピアノ |

#### BGM制御ルール
- **ゾーン遷移：** 2秒のクロスフェードでBGM切替
- **運搬レイヤー：** 遺物を掴んだ瞬間にパーカッションレイヤーをフェードイン（1秒）。離すとフェードアウト
- **天候連動：** 吹雪中はBGM音量 -40%、環境音(風・雪) +20%
- **全滅：** 最後のプレイヤーがHP0 → 現BGMが2秒でフェードアウト → 1秒無音 → 全滅ジングル再生
- **AudioMixer構成：** Master → BGM / SE / Voice / Environment の4グループ。SettingsUIのスライダーと直結

### 15.3 空間オーディオ設定

| パラメータ | 値 | 説明 |
|-----------|-----|------|
| spatialBlend | 0.0 = 2D / 1.0 = 3D | SEテーブルの各SE参照 |
| 3D減衰モデル | Logarithmic Rolloff | Unity AudioSource デフォルト |
| 最小距離（Min Distance） | 2m | この距離以内で最大音量 |

> **AudioMixer ルーティング:**
> ```
> Master (0dB)

> ```

## 16. アニメーションシステム

### 16.1 Animator State Machine

```
                          ┌─────────┐
                     ┌───→│  Death   │───→ [Ragdoll] ───→ [Ghost]
                     │    └─────────┘
┌──────┐  Move   ┌──────┐  Sprint  ┌──────┐
│ Idle │ ←─────→ │ Walk │ ←──────→ │  Run │
└──┬───┘         └──┬───┘          └──┬───┘
   │ Crouch         │ Jump             │
   ▼                ▼                  │
┌──────┐         ┌──────┐  Land    ┌──────┐
│Crouch│         │ Jump │ ───────→ │ Land │ ───→ Idle/Walk
└──────┘         └──┬───┘          └──────┘
                    │ Fall
                    ▼
                 ┌──────┐
                 │ Fall │ ───→ Land / Ragdoll
                 └──────┘

┌──────────┐  Grab   ┌────────────┐
│地上ステート│ ──────→ │ ClimbHold  │ ←→ ClimbMove
└──────────┘  Release └────────────┘

┌──────────┐  Grab   ┌────────────┐
│地上ステート│ ──────→ │   Carry    │ ←→ CarryWalk
└──────────┘  Release └────────────┘

┌──────────┐  Use    ┌────────────┐
│ 任意     │ ──────→ │   Throw    │ ───→ 前のステートへ
└──────────┘         └────────────┘

┌──────────┐  Trigger ┌────────────┐
│ 任意     │ ───────→ │   Emote    │ ───→ 前のステートへ（2秒後 or キャンセル）
└──────────┘         └────────────┘
```

#### ステート一覧

| ステート名 | 遷移条件（入場） | 遷移条件（退場） | ループ |
|-----------|----------------|----------------|-------|
| Idle | MoveSpeed < 0.1 && IsGrounded | MoveSpeed ≥ 0.1 or !IsGrounded | ○ |
| Walk | 0.1 ≤ MoveSpeed < 4.5 && IsGrounded | MoveSpeed変化 or !IsGrounded | ○ |
| Run | MoveSpeed ≥ 4.5 && IsGrounded | MoveSpeed < 4.5 or !IsGrounded | ○ |
| Crouch | IsCrouching && IsGrounded | !IsCrouching | ○ |
| Jump | JumpTrigger | 自動遷移（0.3秒後→Fall） | × |
| Fall | !IsGrounded && verticalVelocity < -2 | IsGrounded（→Land）| ○ |
| Land | IsGrounded（Fallから遷移） | 自動遷移（0.2秒後→Idle/Walk） | × |
| ClimbHold | IsClimbing && MoveSpeed < 0.1 | !IsClimbing or MoveSpeed ≥ 0.1 | ○ |
| ClimbMove | IsClimbing && MoveSpeed ≥ 0.1 | !IsClimbing or MoveSpeed < 0.1 | ○ |
| Carry | IsCarrying（CarryType で BlendTree 切替） | !IsCarrying | ○ |
| Throw | ThrowTrigger | 自動遷移（0.5秒後）| × |
| Death | IsDeadTrigger | → Ragdollシステムへ | × |
| Emote | EmoteTrigger | 2秒後自動復帰 or MoveCancelTrigger | × |
| Ghost | IsGhost | !IsGhost（復活時） | ○ |

### 16.2 Animator パラメータ

| パラメータ名 | 型 | 用途 | 設定元 |
|-------------|-----|------|-------|
| MoveSpeed | Float | 移動速度。Walk/Run遷移、BlendTree制御 | PlayerMovement.cs |
| IsGrounded | Bool | 接地判定 | PlayerMovement.cs（SphereCast） |
| IsClimbing | Bool | クライミング中フラグ | PlayerClimbing.cs |
| IsCarrying | Bool | 遺物運搬中フラグ | PlayerCarrying.cs |
| CarryType | Int | 0=なし/1=片手/2=両手/3=背負い/4=担架 | PlayerCarrying.cs |
| IsCrouching | Bool | しゃがみフラグ | PlayerMovement.cs |
| IsGhost | Bool | 幽霊状態フラグ | GhostController.cs |
| EmoteType | Int | エモート種別（1〜6）。Triggerとして機能 | PlayerController.cs |

> **BlendTree:** Walk/Run は MoveSpeed の値でブレンド（0→Idle, 0〜4→Walk, 4〜7→Run のブレンドツリー）

### 16.3 IK（Inverse Kinematics）仕様

**使用パッケージ：** Animation Rigging

#### クライミング時IK
| 対象 | 設定 |
|------|------|
| 右手/左手 | ClimbingPointのTransformをIK Targetに設定。掴み中は手がポイント位置に吸着 |
| 重み（Weight） | 掴み中 = 1.0 / 未掴み = 0.0。0.2秒のLerpで遷移 |
| 実装 | TwoBoneIKConstraint（上腕→前腕→手） |

#### 遺物持ち上げIK
| 対象 | 設定 |
|------|------|
| 右手/左手 | 遺物のGrabPoint（RelicGrabPoint.cs）のTransformをIK Targetに設定 |
| 両手持ち | 両手のIK Targetを遺物の2つのGrabPointに設定。重み1.0 |
| 片手持ち | 利き手のみIK Weight 1.0。反対手は0.0（自然な状態） |
| 背負い | 両手のIK解除（Weight 0.0）。遺物は背中ボーンにアタッチ |

#### 担架IK
| 対象 | 設定 |
|------|------|
| 2人の持ち手 | 各プレイヤーの両手が担架の端のGrabPointにIK吸着 |
| 高さ追従 | 担架の物理位置に追従。プレイヤーの腕がIKで自動調整 |

#### 足IK
| 対象 | 設定 |
|------|------|
| 接地補正 | Raycastで地面高さを検出。足のIK Targetを地面に合わせて微調整 |
| 傾斜補正 | 傾斜面では足首を地面角度にフィット |
| 重み | IsGrounded時のみWeight 1.0。ジャンプ/落下中は0.0 |

| 背骨（下部） | BoxCollider | 8 | |

#### Ragdoll有効化条件
1. **死亡時：** HP 0到達 → Animator無効化 → 全Ragdollボーンの`isKinematic = false` → 3秒間Ragdoll
2. **大衝撃時（落下速度15m/s以上での壁面衝突）：** 3秒間Ragdoll → 自動復帰
3. **復帰処理：** Ragdoll終了 → 最も地面に近いボーン位置にプレイヤーTransformを移動 → `isKinematic = true` → Animator有効化 → 0.3秒のブレンドで「起き上がり」アニメーション再生

### 17.1 ゲーム全体ステートマシン

```
[Boot] → [Splash] → [TitleScreen] → [MainMenu]
                                        │
                          ┌─────────────┼─────────────────┐
                          ▼             ▼                 ▼
                     [Lobby]     [Cosmetic]          [Settings]

                          ▼
                     [Loading]
                          │
                          ▼

                          │                        │
                          ▼                        │

                     │       │                     │

                          │                        │

### 17.2 各ステートの入場/退場処理

| ステート | 入場処理 | 退場処理 | 遷移先 |
|---------|---------|---------|-------|
| **Boot** | Unity初期化。SaveManager読込。SteamAPI.Init() | スプラッシュ画像表示開始 | → Splash |
| **Splash** | スプラッシュ画像表示（3秒固定）。BGM準備 | フェードアウト | → TitleScreen |
| **TitleScreen** | タイトルBGM再生開始。「Press Any Key」表示 | 任意キー入力でフェードアウト | → MainMenu |
| **MainMenu** | メニューBGM再生。3D背景シーンロード。コスメ適用キャラ表示 | ボタン選択に応じて遷移 | → Lobby / Cosmetic / Settings / Exit |
| **Lobby** | Netcode初期化。ルームコード生成（ホスト時）or コード入力UI。プレイヤーリスト同期開始 | 全員Ready + ホスト「出発！」→ シード生成 → ローディング | → Loading |
| **Loading** | Gameplay.unity 非同期ロード。5レイヤーシード適用。スポーンポイント初期化。TIPS表示（最低3秒） | ロード100%到達 + 3秒経過 | → Basecamp |
| **Basecamp** | ベースキャンプBGM再生。ショップ有効化。天候ボード表示。プレイヤーをスポーン位置に配置 | 出発ゲート通過（全員ゲートエリア内 + ホスト出発インタラクト） | → Expedition |
| **Expedition** | ゾーンBGM開始。天候サイクル開始。ハザード有効化。タイマー（内部のみ）開始 | 帰還成功（ベースキャンプ到達 or ヘリ搭乗）/ 全滅 | → Returning |
| **Returning** | 帰還方法の判定（下山/ヘリ）。ヘリの場合はヘリ演出（2.4参照）。下山の場合はベースキャンプ到達判定 | 帰還演出完了 or 全滅演出完了 | → Result |
| **Result** | リザルトBGM再生。スコア計算。称号判定。コスメ解放判定。6段階演出再生（14.6参照）。データ保存 | 「メインメニューへ」ボタン押下 | → MainMenu |

|------|------|
| ポーズキー | Escキー / Menu ボタン |

## 18. セーブ/データ永続化

### 18.1 保存対象

| データカテゴリ | 保存先 | 内容 |

### 20.3 パフォーマンス予算

| 項目 | 上限値 | 制約理由 |
|------|-------|---------|
| アクティブ Rigidbody（物理演算中） | 200個 | FixedUpdateの負荷。ホスト権威でホストPCの負荷が支配的 |

| テクスチャVRAM | 1.5 GB以下 | 最低GPU VRAM 4GBの37% |
| ネットワーク帯域 | 256 Kbps/クライアント | ホスト型P2Pの帯域制約 |
| 物理同期パケット | 20Hz | 50msごとに物理オブジェクト位置同期 |

> **最適化指針：**
> - ロープの物理セグメント50個が最大の負荷源。ロープ4本同時使用（200リジッドボディ）を上限値の根拠とする
> - 視界外のRigidbodyはスリープ状態を維持（自動。Unity Physics デフォルト動作）
> - LOD: ローポリのためLOD不要。距離によるカリングのみ

## 25. 競合分析＆差別化

| 比較軸 | ccc（本作） | Cairn | R.E.P.O. | Lethal Company | Content Warning |
|--------|-----------|-------|---------|---------------|----------------|

### 差別化ポイント
1. **「登り簡単 × 運搬カオス」のハイブリッド：** Cairnの高難度を避けつつ、R.E.P.O.の物理カオスをクライミングに適用
